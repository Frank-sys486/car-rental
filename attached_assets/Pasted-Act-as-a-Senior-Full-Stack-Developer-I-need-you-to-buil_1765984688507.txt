PROJECT ARCHITECTURE: "Simulation Mode"
This app is designed for a single business owner who acts as both the Admin and the Customer interface.

TECH STACK (Strict):
- Frontend: React (Vite) + Tailwind CSS.
- Backend Option A (Production): Supabase (Auth/Database).
- Backend Option B (Local Dev): Local Mock Data (JSON).
- Router: React Router DOM.
- Icons: Lucide React.
- Charts: Use a lightweight library or CSS Grid for the Gantt Chart.

CRITICAL REQUIREMENT: OFFLINE/LOCAL DEVELOPMENT
You must architect the data layer to support a "Mock Mode" for local development without Supabase.
1. Create a `services/api.js` abstraction layer.
2. Check an environment variable (e.g., `VITE_USE_MOCK=true`).
3. If true, return data from a local `mockData.json` file.
4. If false, fetch data from Supabase.
This allows the developer to run `npm run dev` and test the full flow immediately without setting up the database.
